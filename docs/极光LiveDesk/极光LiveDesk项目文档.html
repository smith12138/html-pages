一、产品概述与定位

极光(Aurora Mobile)推出的 SaaS 客服产品，当前用于 TT在线客服 项目

1.1 产品简介

EngageLab LiveDesk 是极光（Aurora Mobile）旗下的全渠道客服平台。它同时支持 AI Agent 和 Human Agent 两种应答模式，客户通过 Web LiveChat Widget 等渠道发起会话后，系统会根据配置决定由 AI 自动应答还是转交人工处理。

基本信息

服务模式AI Agent 自动应答 + Human Agent 人工处理
当前项目TT在线客服
访问地址https://www.engagelab.com/console/livedesk/

消息展示

用户消息显示在左侧灰色气泡，客服消息在右侧蓝色/紫色气泡，AI回复会带一个 AI Badge 标识。每条消息都标注了发送时间。

系统内置了自动翻译功能，提供 MT-Flash（快速）和 MT-Plus（高质量）两种翻译模型，支持双向互译，单次限制 1000 字符，可配置自定义术语表（JSON 格式）。当前项目主要用于 PT（葡萄牙语）和 CN（中文）之间的互译。每条消息旁提供"翻译"按钮，点击即可查看对应语言的译文。另外还支持 Typing Indicator（输入指示）和 Read Receipts（已读回执）。

用户名称采用系统自动生成的动物组合名，格式为"形容词-动物"，如 dynamic-skunk、excess-aphid、flat-bison 等，保护用户隐私的同时便于客服辨识不同对话。

工具对应操作

链接插入超链接

其他编辑功能

  • "/" 快捷回复：在富文本编辑器中输入 "/" 即可调出预设的回复模板列表，支持快速搜索和插入常用回复
  • 附件上传：支持图片和文件
  • 内部备注模式：切换到 Internal Note 后，消息仅团队内可见，不会发送给客户
  • 发送快捷键：可在偏好中设为 Enter 或 Cmd+Enter

官方核心指标

指标说明
对话数量（Conversation Count）各时间段的新建对话总数
收到消息数（Messages Received）客户发送的消息总量
发出消息数（Messages Sent）客服和AI回复的消息总量

客户等待时间（Customer Wait Time）客户等待客服回复的累计时长

管理功能

高级功能（官方文档补充）

  • 联系人合并（Contact Merge） -- 支持将重复的联系人记录合并，合并时会自动整合所有属性信息和关联的对话历史

11.2 AI 能力详细说明（官方文档补充）

问题分类（Issue Classification）

AI 自动识别用户消息内容，通过 GPTBots Agent API 匹配到预设的问题分类类别。启用后系统会对每条对话自动打上分类标记，便于统计和路由。

优化客服消息（Message Optimization）

支持自定义 AI 指令（Custom Instructions），对客服发送的回复内容进行实时优化建议，提升客服回复的质量和专业度。

解决置信度（Resolution Confidence）需订阅

AI 质量检测功能，对已标记为 Resolved 的对话进行置信度评估，判断问题是否真正被解决。此功能为订阅制付费功能。

FCR 识别（First Contact Resolution）需订阅

首次联系解决率识别。系统在对话解决后的 24 小时内持续监控，评估该对话是否在首次联系中即完成解决。此功能为订阅制付费功能。

自动推断解决识别 需订阅

针对系统自动解决（auto-resolved）的对话，AI 会进一步评估该对话是否被真正解决，避免误判。此功能为订阅制付费功能。

对话消息翻译

提供两种翻译模型：MT-Flash（快速翻译）和 MT-Plus（高质量翻译）。单次翻译限制 1000 字符。支持自定义术语表（JSON 格式），可配置专业术语的翻译对照。翻译支持双向互译，适用于跨语言客服场景。

17.2 快捷回复（Canned Responses）

快捷回复通过在输入框输入 "/" 触发（shortcode 模式）。系统会弹出预设回复模板列表，支持关键词搜索快速定位。管理员可预先配置常用回复模板，客服选择后一键插入编辑器。

17.3 宏操作（Macros）

Macros 允许将多个操作组合为一键执行的操作集。支持 公开（Public） 和 私有（Private） 两种可见性：公开宏对所有客服可见，私有宏仅创建者可用。常见操作组合包括：分配客服 + 添加标签 + 变更状态等。

17.4 Dyte 音视频集成

LiveDesk 集成了 Dyte 音视频通话功能，支持在对话中发起实时视频或语音通话。配置时需要填入 Dyte 的 Organization ID 和 API Key。启用后客服可在对话窗口直接与客户进行语音/视频沟通，适合需要面对面交流的复杂场景。

十八、Widget SDK 接入（官方文档补充）

JavaScript SDK 提供完整的 Widget 控制能力，支持用户身份绑定、自定义属性、界面控制和事件监听

18.1 全局配置（livedeskSettings）

通过全局变量 window.livedeskSettings 进行 Widget 初始化配置，支持 20+ 个参数，包括：渠道 Token、语言、位置、颜色主题、暗色模式、气泡样式、自定义属性等。

window.livedeskSettings = { token: 'YOUR_CHANNEL_TOKEN', locale: 'pt_BR', position: 'right', type: 'expanded_bubble', darkMode: 'auto', // ...20+ 可配置参数 };

18.2 JavaScript SDK 方法

方法功能说明
setUserId(userId)绑定用户身份标识，支持普通模式和 HMAC 加密模式（防伪造）
setCustomAttributes(attrs)设置自定义对话或用户属性
toggle()切换 Widget 的展开/收起状态
toggleBubbleVisibility()切换气泡按钮的显示/隐藏
popoutChatWindow()将聊天窗口弹出为独立窗口
setLocale(locale)动态切换 Widget 显示语言

18.3 事件监听（Event Listeners）

事件名称触发时机
livedesk:readyWidget 加载完成并准备就绪时触发
livedesk:on-message收到新消息时触发
livedesk:errorWidget 发生错误时触发

HMAC 加密用户身份（安全模式）

为防止用户身份伪造，setUserId 支持 HMAC 加密模式。服务端使用 API Secret 对 userId 进行 HMAC-SHA256 签名，将签名结果连同 userId 一起传入 SDK，系统会自动验证签名有效性。
