5.1 用户管理

系统用户的全生命周期管理，包括创建、配置、权限分配与禁用。

  • 创建新用户：填写基本信息，指定角色与初始权限

  • 用户状态管理：启用/禁用账户

RBAC 说明：系统采用基于角色的访问控制（Role-Based Access Control）模型，每个角色绑定一组预定义权限集合。用户通过角色继承对应权限，避免逐用户配置的维护成本。当前共 5 种系统角色（含 1 种超级管理角色）。

七、核心业务概念

平台涉及的关键术语与业务概念释义
术语英文释义
BMBusiness ManagerMeta 的商务管理平台资产管理单元，是广告账户、主页、像素等资产的顶层容器。每个 BM 可包含多个广告账户。
广告账户Ad AccountBM 下的独立广告投放单元，每个广告账户拥有独立的预算、花费记录和投放数据。
投手（Shooter）Ad Operator广告投放操作员角色，负责日常广告系列的创建、优化、预算调整等操作。占系统用户总数的 79%。
组长（Leader）Team Leader团队负责人角色，管理一组投手，负责账户分配、投放策略指导和团队绩效管理。
美术（Art）Creative Designer创意设计角色，负责广告素材（图片、视频、文案）的制作与维护。
API 限额API Rate LimitFacebook 对每个广告账户的 API 调用频率限制。超出限额将导致接口返回错误，影响投放操作。
账户健康度Account HealthFacebook 对广告账户的质量评分体系，健康度低的账户可能面临投放限制甚至封停。
三元组导出Triplet ExportAI 模型训练数据的特定格式，以三元组（Anchor, Positive, Negative）形式组织，用于训练广告效果预测模型。
回填数据Backfill Data对历史时间段内缺失或不完整数据的补充导出，确保训练数据的时间连续性。
优质数据Quality Data经过筛选和标注的高质量训练样本，用于模型精调以提升预测准确率。
